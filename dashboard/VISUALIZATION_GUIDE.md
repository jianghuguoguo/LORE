# 🚀 Evo-Pentest 智能检索可视化系统

## 系统概述

本系统集成了 **System 2 Reflection** 机制，为渗透测试知识库提供自适应检索能力。系统能够：

1. **自动检测**检索失败（Failure Detection）
2. **智能诊断**失败原因（Reflection Diagnosis）
3. **生成优化**查询（Query Rewriting）
4. **循环迭代**直到获得高质量结果

---

## 🎯 核心功能

### 1. 智能检索面板

位于 Dashboard 的 **"智能检索"** Tab，提供：

- **实时流程可视化**：展示 Search → Detect → Diagnose → Rewrite 的完整循环
- **迭代历史追踪**：记录每一轮查询的演化过程
- **效用评分展示**：可视化文档的实战价值（Utility Score）
- **动态动画效果**：科技感十足的交互体验

### 2. 技术架构

```
用户查询
   ↓
[Search] 向量检索
   ↓
[Detect] 失败检测（5大启发式规则）
   ↓
[Diagnose] 诊断失败类型
   ↓
[Rewrite] 生成3个候选查询
   ↓
循环（最多3次）
   ↓
最终结果 + Utility Scorer 重排序
```

### 3. 失败检测规则

系统使用以下启发式规则判断检索质量：

1. **结果为空**：返回文档数 = 0
2. **文档过短**：平均长度 < 50字符
3. **缺少实战关键词**：无 "Payload", "Exploit", "POC", "Script" 等
4. **相关度过低**：Top-3 平均分 < 0.4
5. **效用分数低**：Cross-Encoder 评分 < 0.3

### 4. 诊断类型

- **Query Drift**（查询漂移）：使用CVE编号但文档用产品名
- **Granularity Imbalance**（粒度失衡）：查询太具体或太宽泛
- **Thesaurus Gap**（同义词缺失）：中英文术语不匹配
- **Domain Mismatch**（领域不匹配）：攻击类型错位

## 📖 使用指南

### 启动服务

```bash
cd dashboard
python app.py
```

访问：`http://localhost:5000`

### 操作步骤

1. **进入智能检索 Tab**
   - 点击顶部导航栏的 "🧠 智能检索"

2. **输入查询**
   - 在 "检索查询" 框中输入，例如：
     - `CVE-2017-10271 漏洞利用`
     - `SQL注入绕过WAF payload`
     - `Linux提权技巧`

3. **配置迭代次数**
   - 选择最大迭代次数（1-3次）
   - 建议选择 3 次以获得最佳效果

4. **启动检索**
   - 点击 "🚀 启动智能检索" 按钮

5. **观察可视化**
   - **流程步骤**：实时展示当前执行的阶段
   - **迭代历史**：查看每轮的诊断和重写结果
   - **效用评分**：查看最终文档的实战价值

## 🎨 可视化特性

### 1. 流程步骤动画

- **激活状态**：当前步骤会有霓虹灯脉冲效果
- **成功标志**：绿色渐变表示检测成功
- **失败标志**：红色渐变表示需要重写
- **进度条**：动态展示整体进度

### 2. 迭代卡片

每个迭代显示：
- **迭代编号**：第几轮
- **查询内容**：当前使用的查询语句
- **诊断结果**：失败原因和根本原因
- **重写查询**：生成的候选查询（可点击查看）
- **性能指标**：结果数量、耗时

### 3. 效用评分条

- **绿色**（High）：分数 ≥ 0.7，高价值文档
- **橙色**（Medium）：0.4 ≤ 分数 < 0.7，中等价值
- **红色**（Low）：分数 < 0.4，低价值文档

### 4. 统计仪表板

展示：
- **总迭代次数**
- **总耗时**（毫秒）
- **平均耗时/轮**
- **成功率**（百分比）

## 🛠️ 技术栈

### 后端
- **Flask**：Web 服务器
- **Processors**：自适应检索核心逻辑
  - `failure_detector.py`：失败检测器
  - `reflection_diagnoser.py`：诊断器
  - `query_rewriter.py`：查询重写器
  - `adaptive_retriever.py`：自适应检索控制器

### 前端
- **Vanilla JavaScript**：交互逻辑
- **CSS3 动画**：霓虹灯、渐变、过渡效果
- **Font Awesome**：图标库

### 样式特色
- **赛博朋克风格**：蓝紫色科技渐变
- **霓虹灯效果**：`neon-pulse` 动画
- **流畅过渡**：渐入、缩放、阴影动画

## 🔧 高级定制

### 调整动画速度

编辑 `adaptive.js`：

```javascript
this.animationDelay = 800; // 毫秒，调整步骤动画间隔
```

### 修改配色方案

编辑 `adaptive.css`：

```css
/* 主题色 */
--primary-color: #667eea;    /* 主色调 */
--secondary-color: #764ba2;  /* 次色调 */
--accent-color: #00ffff;     /* 强调色（霓虹青色）*/
```

### 扩展诊断类型

编辑 `processors/reflection_diagnoser.py`，添加新的诊断规则。

## 🐛 故障排除

### 显示"自适应检索模块未安装"

**原因**：processors 模块导入失败

**解决方案**：
```bash
# 检查模块是否存在
ls processors/adaptive_retriever.py

# 安装依赖
pip install -r requirements.txt
```

### 前端样式错乱

**原因**：CSS 文件未正确加载

**解决方案**：
1. 检查 `dashboard/static/css/adaptive.css` 是否存在
2. 清除浏览器缓存（Ctrl+F5）
3. 检查浏览器控制台的错误信息

### API 返回 500 错误

**原因**：后端逻辑异常

**解决方案**：
1. 查看 Flask 控制台的错误堆栈
2. 检查 `processors/` 模块是否完整
3. 确认 Python 版本 ≥ 3.8
