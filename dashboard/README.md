# 爬虫管理系统 Web 界面

## 📋 简介

这是一个专为多源爬虫系统设计的 Web 管理界面，提供了直观的可视化操作面板，让您可以轻松管理和监控所有爬虫。

## ✨ 功能特性

### 🎮 控制面板
- **数据源选择**: 可视化选择要启用的爬虫数据源
- **参数配置**: 设置搜索关键词、爬取页数等参数
- **一键启停**: 快速启动和停止爬取任务

### 📊 实时监控
- **进度跟踪**: 实时显示每个数据源的爬取进度
- **系统日志**: 彩色日志输出，清晰展示系统运行状态
- **数据统计**: 可视化展示各数据源的爬取数量

### 📁 数据管理
- **结果浏览**: 在线查看最新的爬取结果
- **文件列表**: 管理已保存的所有数据文件
- **自动保存**: 爬取完成后自动保存到 raw_data 目录

## 🚀 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 启动服务

#### Windows
```bash
cd dashboard
python app.py
```

#### Linux/Mac
```bash
cd dashboard
python3 app.py
```

### 3. 访问界面

打开浏览器访问: **http://localhost:5000**

## 📖 使用指南

### 基本操作流程

1. **选择数据源**
   - 在左侧控制面板中勾选要使用的爬虫
   - 可以使用"全选"按钮快速选择所有数据源

2. **配置参数**
   - 输入搜索关键词（例如：渗透测试、CVE-2024-23897）
   - 设置最大爬取页数（建议 1-10 页）

3. **开始爬取**
   - 点击"开始爬取"按钮
   - 系统会自动切换到进度监控面板
   - 实时显示每个数据源的爬取进度

4. **查看结果**
   - 切换到"爬取结果"标签查看最新数据
   - 切换到"文件管理"标签查看历史文件

## 🔧 API接口

Web界面提供了以下RESTful API接口：

| 接口 | 方法 | 说明 |
|-----|------|------|
| `/api/crawlers` | GET | 获取所有爬虫信息 |
| `/api/status` | GET | 获取当前运行状态 |
| `/api/start` | POST | 启动爬取任务 |
| `/api/stop` | POST | 停止爬取任务 |
| `/api/logs` | GET | 获取系统日志 |
| `/api/results` | GET | 获取爬取结果 |
| `/api/files` | GET | 列出已保存文件 |
| `/api/config` | GET | 获取配置信息 |

### 示例：启动爬取

```bash
curl -X POST http://localhost:5000/api/start \
  -H "Content-Type: application/json" \
  -d '{
    "sources": ["csdn", "github"],
    "query": "渗透测试",
    "max_pages": 5
  }'
```

## 📝 配置说明

### 端口配置

默认端口：`5000`

如需修改，编辑 `dashboard/app.py` 文件：

```python
app.run(
    host='0.0.0.0',
    port=5000,  # 修改此处
    debug=True
)
```

### 更新频率

- 普通状态: 每 3 秒更新一次
- 爬取中: 每 1 秒更新一次

可在 `static/js/main.js` 中修改：

```javascript
// 普通更新
setInterval(() => { ... }, 3000);  // 3秒

// 快速更新（爬取时）
setInterval(() => { ... }, 1000);  // 1秒
```

## 🔍 故障排查

### 无法启动服务

1. 检查是否安装了 Flask：
```bash
pip install Flask flask-cors
```

2. 检查端口是否被占用：
```bash
# Windows
netstat -ano | findstr :5000

# Linux/Mac
lsof -i :5000
```

### 爬虫无法启动

1. 检查爬虫配置文件 `crawlers/config.py`
2. 确保相关爬虫已启用：`ENABLED_SOURCES`
3. 查看实时日志获取错误信息

### 数据未显示

1. 切换到"实时日志"标签查看错误信息
2. 检查 `raw_data` 目录权限
3. 确保网络连接正常
