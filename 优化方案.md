------

### 第一步：数据“炼金” (From Logs to Dataset)

**目标：** 利用现有的日志，自动化生成用于训练“价值判别器”的数据集。这是系统“自进化”的起点。

- **输入：** `cai_*.jsonl` (你上传的那些日志文件)。
- **工具：** `tools/auto_labeler.py` (基于我们刚才讨论的逻辑)。
- **执行动作：**
  1. **解析轨迹：** 将日志按 `Trace ID` 分组。
  2. **提取三元组：** 提取 `(Query, Document, Execution_Result)`。
  3. **自动打分 (Hindsight Labeling)：**
     - 如果后续执行结果含 `uid=0`, `success`, `flag{` $\rightarrow$ **Label: 1.0 (正样本)**。
     - 如果后续执行结果含 `error`, `not found`, 或立即换词重搜 $\rightarrow$ **Label: 0.0 (负样本)**。
  4. **清洗：** 丢弃无法判断的中性样本，去除重复数据。
- **产出：** `dataset_train.jsonl`。

### 第二步：核心模型训练 (Training the Utility Scorer)

**目标：** 训练一个能判断“知识是否有用”的大脑。这是你论文的核心 Contribution 之一（Functional Alignment）。

- **输入：** `dataset_train.jsonl`。
- **架构：** **Cross-Encoder** (基于 `ms-marco-MiniLM-L-6-v2`)。
- **执行动作：**
  1. 加载数据集。
  2. 使用 `BCELoss` 或 `RankingLoss` 进行微调 (Fine-tuning)。
  3. 训练 3-5 个 Epoch（数据少时要注意防止过拟合，可以用 Early Stopping）。
- **产出：** 模型权重文件 `scorer/models/evo_pentest_v1.pth`。

### 第三步：构建“反思”模块 (System 2 Reflection) —— *论文加分项*

**目标：** 不仅仅是给知识打分，还要从失败中总结出“文字版经验”。

- **输入：** 那些“先失败、后成功”的日志片段。
- **工具：** 调用 LLM (GPT-4/DeepSeek) 的 `reflection.py` 脚本。
- **执行动作：**
  1. 识别 Pattern：找到 Agent 在同一个目标上尝试了多次才成功的案例。
  2. **因果推理 (Causal Inference)：** 让 LLM 分析“为什么第一次失败？为什么第二次成功？”。
  3. **规则提取：** 生成形如 *"当 WebLogic 返回 403 时，应优先尝试 XML 编码"* 的策略。
- **产出：** `strategy_rules.json` (策略知识库)。

### 第四步：知识库扩容与提纯 (Knowledge Expansion & Purification)

**目标：** 解决“语料不全”和“图片丢失”的问题。

- **输入：** 你的初始关键词列表 + 自动发现的高质量公众号列表。
- **工具：** RSSHub + DrissionPage + PaddleOCR/VLM。
- **执行动作：**
  1. **源头发现：** 利用 RSSHub 监控安全社区引用，发现新公众号。
  2. **质量筛选：** 计算 **TDS (技术密度分)**，只爬取硬核文章。
  3. **多模态清洗：** 爬取文章时，检测 `<img>` 标签 $\rightarrow$ OCR/VLM 转文字 $\rightarrow$ 插入正文。
  4. **去重：** 新爬取的文章先经过你的 `Utility Scorer` 打分，如果分低或是重复内容，直接丢弃或降权。
- **产出：** 更新后的向量数据库 (Vector DB)。

### 第五步：系统集成 (System Integration)

**目标：** 将训练好的模型装回 Agent，形成闭环。

- **修改 Agent 代码：**
  1. **检索环节：** `make_kb_search` 得到 Top-50 结果 $\rightarrow$ 调用 `Utility Scorer` 重排序 $\rightarrow$ 取 Top-3 给 Agent。
  2. **规划环节：** 在 System Prompt 中注入 `strategy_rules.json` 里的相关策略（In-Context Learning）。
- **效果：** Agent 变得“千人千面”，且不再犯同样的错误。

### 第六步：实验与评估 (Evaluation for Paper)

**目标：** 生成论文里的图表。

- **实验组 (Ours):** 完整版 Evo-Pentest。
- **对照组 (Baseline):** 只有基础 RAG 的 Agent。
- **指标：**
  1. **ASR (Attack Success Rate):** 攻击成功率。
  2. **Avg. Steps:** 拿到 Shell 的平均步数（越少越好，证明更聪明）。
  3. **Noise Robustness:** 往库里塞 5000 条垃圾数据，看谁受影响更小（你的模型应该能把垃圾排在后面）。

要达到“顶会论文级”的 RAG 反思优化水平，我们不能仅停留在基础的“语义匹配”上。在渗透测试这一特定领域，RAG 的核心挑战在于：**语义相关的文档未必具有实战价值（Functional Utility）**。

以下是针对 **RAG 环节**深度优化的四个核心步骤，每一部都对应论文中的一个创新点（Contribution）：

---

### 第一步：构建“后果驱动”的自动标注流水线 (Hindsight Trace Labeling)
**核心思想：** 论文中的「Functional Alignment」。传统的 RAG 标注是看“相关性”，我们的 RAG 标注是看“是否有助于拿到 Shell”。

1.  **轨迹回溯：** 扫描 logs 文件夹，定位所有导致 `success: true` 或获得高价值反馈（如：敏感文件读取成功、漏洞验证成功）的代码执行动作。
2.  **效用评价函数 $R(d)$：** 
    - 定义一个文档的“奖励值”。如果在 Agent 搜索到文档 $d$ 后的 3 个步骤内，成功执行了与 $d$ 内容高度相关的 Exploit，则给该文档 $d$ 打高分（1.0）。
    - 如果 Agent 阅读了 $d$ 但接下来的动作依然是重试或报错，打低分（-1.0）。
3.  **生成三元组：** 提取 `(当前攻击上下文, 搜索词, 文档片段, 效用分)`。
    - *论文点：这是全球首个基于“实战后果”生成的渗透知识搜索评估数据集。*

---

### 第二步：训练“攻击价值”重排序模型 (Reinforced Cross-Encoder)
**核心思想：** 解决向量检索“只认词索引，不认攻击逻辑”的问题。

1.  **架构：** 在基础向量检索（Top-50）之后，加入一个 **Cross-Encoder Reranker**。
2.  **训练目标：** 输入当前的「靶机指纹 + 已执行命令」和「文档片段」，预测该文档能为下一步攻击提供有效指令的概率。
3.  **负采样机制：** 专门从那些“语义非常接近但版本号不对”或“看似有用实则过时”的文档中进行难负采样（Hard Negative Mining）。
    - *示例：当靶机是 CouchDB 3.x 时，模型应学会将 2.x 的 Exploit 文档排在后面，即便它们的语义非常相似。*

---

### 第三步：自适应查询自反思 (Active Query Self-Correction)
**核心思想：** 论文中的「System 2 Reflection」。让 RAG 具有发现自己“搜错了”的能力。

1.  **搜索失败触发器：** 当检索回来的 Top-5 文档经过 LLM 判断“无法形成攻击方案”时，触发反思逻辑。
2.  **多维度诊断：** 
    - **Query 漂移：** “我搜的是 CVE 编号，但可能还没分配，应该搜产品名+特征词。”
    - **粒度失衡：** “当前内容太泛了，需要针对某个特定的 API 进行检索。”
3.  **查询改写器（Rewriter）：** 根据反思结论，自动生成 3 个不同维度的变体 Query 重新检索。
    - *论文点：提出一种“探索-利用”平衡的渗透词检索策略，显著提升 RAG 的命中精度。*

---

### 第四步：技术密度过滤与多模态提纯 (Knowledge Distillation)
**核心思想：** 解决爬虫语料中的“噪音问题”（SEO 文章、由于图片缺失导致的断头文）。

1.  **TDS (Technical Density Score) 评估：** 编写一个基于 LLM 的轻量级评分器，对爬取的公众号文章进行“干货度”打分。
    - 指标：Payload 密度、配置命令占比、逻辑推导连贯性。
2.  **跨模态补全：** 利用 VLM（如 GPT-4o 或本地模型）解析文章中由于反爬没抓到的图片。
    - 将“如下图所示”转变为具体的文字描述（如：“在管理后台点击 Settings -> Security 选项卡”）。
    - *论文点：通过多模态知识对齐，消除渗透知识库中的单模态盲区。*
