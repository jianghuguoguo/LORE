### 1.2 开发背景
随着网络安全风险与漏洞利用技术的快速演进，渗透测试活动产生了大量高价值实战日志与过程数据。传统方式对经验的沉淀依赖人工复盘，存在效率低、标准不统一、难复用、难持续更新等问题。

因此，我们希望构建一个能够从真实攻防会话中自动抽取经验、结构化整理知识、持续融合优化并回流检索系统的工程化平台。LORE 系统围绕这一目标，形成“日志标准化 -> 经验蒸馏 -> 跨会话融合 -> 缺口补全 -> 知识回流”的闭环能力，提升渗透测试知识资产的复用效率与实战可迁移性。

### 1.3 定义
- 系统、本系统、该系统：指 LORE 渗透测试知识蒸馏系统。
- 用户：指使用本系统进行部署、运行、分析、管理的人员，包括安全研究员、渗透测试工程师、平台运维人员。
- 会话日志：指渗透测试执行过程中产生的原始 JSONL 日志。
- 五层流水线：指 Layer0、Layer1、Layer2、Layer3、Layer4 的分阶段处理链路。
- 五类知识：FACTUAL、PROCEDURAL_POS、PROCEDURAL_NEG、METACOGNITIVE、CONCEPTUAL。
- KLM：Knowledge Lifecycle Management，指融合后知识条目的生命周期管理。
- RAGFlow 回流：指将融合后的经验条目上传至 RAG 检索知识库供智能检索调用。

## 2、系统概述

### 2.1 系统用途
本系统用于将渗透测试原始会话数据自动转化为可检索、可解释、可复用的结构化安全经验，主要用途如下：

1. 安全经验沉淀：自动抽取成功路径、失败根因、事实证据和策略规则。
2. 知识库建设：将零散日志升级为标准化知识条目，形成可持续增长的经验库。
3. 过程分析与复盘：提供会话维度分析和跨会话融合能力，降低人工复盘成本。
4. 缺口驱动补全：针对知识盲区执行定向爬取与外部知识同步。
5. 检索增强支持：将高价值经验同步到 RAG 知识库，服务下游智能问答与决策支持。

界面展示建议：
- 建议展示“总览看板 + 会话浏览”两个页面，体现系统用途覆盖统计、分析、复盘。

### 2.2 系统功能及特点
本系统具备以下功能模块与技术特点：

1. 五层自动化蒸馏流水线
- Layer0：日志标准化与会话拆分。
- Layer1：LLM 会话标注（结果、失败根因等）。
- Layer2：经验提炼与五类知识生成。
- Layer3：跨会话融合、冲突处理、KLM 注册。
- Layer4：经验缺口识别与定向补全。

2. 可视化管理看板
- 支持总览统计、经验分类浏览、会话分析、流水线状态跟踪。
- 支持经验详情弹窗查看证据、规则、建议、标签与元数据。

3. 爬虫与知识补全
- 支持网站源爬虫、微信公众号爬虫、RSS 自动订阅。
- 支持外部知识库同步与原始数据管理。

4. 知识健康与缺口治理
- 展示冲突条目、生命周期分布、RAGFlow 同步状态。
- 依据缺口评分触发定向关键词爬取。

5. 工程化与可扩展性
- 模块化设计，支持脚本化运行与可视化运维双模式。
- 支持分阶段运行和全流程运行，便于生产与测试场景切换。

- 操作系统：Windows 10/11、Linux、macOS
- Python：3.10 及以上
- 包管理：pip
- 浏览器：Chrome/Edge 最新版本（用于 Dashboard）
- 关键依赖：Flask、flask-cors、openai、apscheduler、beautifulsoup4、lxml、trafilatura 等

### 3.2 系统安装

步骤 1：获取源代码

```bash
git clone <repository-url>
cd Evo-PentestRAG-main
```

步骤 2：创建并激活虚拟环境

Windows：

```bash
python -m venv .venv
.venv\Scripts\activate
```

Linux/macOS：

```bash
python3 -m venv .venv
source .venv/bin/activate
```

步骤 3：安装依赖

```bash
pip install -r requirements.txt
```

步骤 4：配置参数
- 编辑 configs/config.yaml，填写 LLM 与 RAGFlow 相关配置。
- 如需增强采集能力，可按需配置 crawlers 相关参数。

步骤 5：启动 Dashboard

```bash
cd dashboard
python app.py
```

浏览器访问：http://localhost:5000

步骤 6：运行流水线（可选）

```bash
cd ..
python run/run_full_pipeline.py
```

或执行交互式入口：

```bash
python lore.py
```

安装验证标准：
- Dashboard 页面正常打开；
- 总览统计可加载；
- 能正常触发流水线或爬虫任务；

## 4、软件功能

### 4.1 功能入口与整体交互
系统采用“顶部状态栏 + 左侧功能导航 + 主工作区”的工作台式布局，用户进入系统后可立即看到经验总量、核心知识层计数、会话数量以及流水线运行状态。顶部提供“开启反思”快捷触发按钮，用于在不切换页面的情况下直接发起核心蒸馏流程；右上角提供深浅主题切换，满足不同展示场景下的阅读需求。左侧导航按照“经验库、分析、融合、知识库”分区组织，能够清晰体现系统并非单一页面，而是覆盖采集、提炼、治理、回流的完整业务链路。

界面截图建议：截取 Dashboard 首屏全景，需同时包含顶部状态栏、左侧导航分区和主内容区标题，作为“系统完整性证明图”（对应附录A“系统首页全景”项）。

### 4.2 总览看板与统计分析
总览看板用于向评审人员直观展示系统的“数据规模 + 结构分布 + 运行健康度”。页面将经验总量、FACTUAL、PROCEDURAL_NEG、METACOGNITIVE、CONCEPTUAL、会话数等指标统一呈现，并通过知识层分布、会话结果分布、置信度分布、目标服务分布、攻击阶段分布、提取来源分布等图表展示知识资产质量。该页面既可作为日常运维首页，也可作为软著鉴定时证明“系统具备可视化分析与统计能力”的关键证据页。

界面截图建议：建议至少提供两张图，第一张突出顶部统计卡片与知识层分布图，第二张突出会话结果分布、置信度分布、服务分布等图表区域（对应附录A“总览看板相关项”）。

### 4.3 五类经验库浏览与检索
系统将经验按 FACTUAL、PROCEDURAL_POS、PROCEDURAL_NEG、METACOGNITIVE、CONCEPTUAL 五类进行独立分页展示。每类页面均支持关键词检索、分页浏览、快速刷新，经验卡片展示 exp_id、置信度、目标服务、CVE 标签、提取来源和摘要信息，用户可在同一交互范式下快速定位事实证据、成功路径、失败避坑和策略规则。该模块的核心价值在于把原本混杂在日志中的知识转化为可查询、可筛选、可持续复用的知识实体。

界面截图建议：建议覆盖 FACTUAL、PROCEDURAL_POS、PROCEDURAL_NEG、METACOGNITIVE/CONCEPTUAL 四类页面，且每张图需同时保留搜索框、分页和经验卡片字段（对应附录A“经验列表相关项”）。

### 4.4 经验详情弹窗与结构化解释
用户点击任一经验卡片后，系统弹出统一详情窗口，先展示元数据（来源会话、结果、BAR 分值、置信度、提取方式、目标服务、CVE、版本、创建时间），再按知识层展示专属内容。FACTUAL 提供“发现事实 + CVE Context + 原始证据”；PROCEDURAL_NEG 提供“失败命令、失败模式、决策规则、修复建议”；PROCEDURAL_POS 提供“参数化命令模板、原始命令、成功命令、成功证据、前置条件、后续动作”；METACOGNITIVE 与 CONCEPTUAL 则重点展示经验教训、最优路径、核心洞察与触发条件。该模块用于证明系统具备“可解释 AI 经验输出”能力，而不是仅给出不可追溯结论。

界面截图建议：建议分别截取 FACTUAL、PROCEDURAL_POS、PROCEDURAL_NEG、METACOGNITIVE/CONCEPTUAL 的详情弹窗，重点保留“原始证据、命令模板、失败命令、决策规则、核心洞察”等关键区域（对应附录A“详情弹窗相关项”）。

### 4.5 会话浏览与复盘分析
会话浏览页以 Session 为中心组织数据，展示目标服务、CVE、攻击结果、BAR 评分、经验总量与分层分布。相较于经验库的“条目视角”，本页提供“过程视角”，便于用户回看单次攻防的知识产出质量、失败集中区域和经验覆盖完整度。对于鉴定材料，本页可证明系统支持从“条目管理”进一步上升到“会话复盘”的分析能力。

界面截图建议：截取会话列表整页，需包含分层分布列、结果标签与 BAR 评分列（对应附录A“会话浏览项”）。

### 4.6 分阶段流水线运行与实时日志
流水线页面提供 Layer0 至 Upload 的分阶段执行能力，支持全选/全不选、跳过上传、详细日志、状态重置，并实时展示各阶段状态、耗时与当前执行步骤。用户既可通过该页精细控制执行范围，也可通过顶部“开启反思”按钮快速触发常用链路，满足“研究调试”和“批量生产”两类场景。页面中的实时日志终端用于记录每次运行的全过程，支持运维排障与审计留痕。

界面截图建议：建议提供“阶段选择与控制区”“运行中日志终端”“运行完成状态回显”三张连续图，形成完整运行证据链（对应附录A“流水线相关项”）。

### 4.7 爬虫管理：公众号、网站、RSS、外部知识库与原始数据治理
爬虫管理页是系统补充外部知识的重要入口，功能覆盖“采集、同步、清理”全生命周期。微信公众号区域支持种子账号选择、全选/全不选、账号新增与移除、搜狗直采与原生微信双模式切换、代理与 mitmdump 联调、原生状态自检以及文章预览；网站爬虫区域支持按关键词/CVE 定向爬取并可选择多数据源；RSS 区域支持自动订阅状态展示与手动立即同步；外部知识库同步区域支持按仓库选择性同步并保留同步状态；RAG 源文件管理区域支持按来源、按文件、按全量级别清理数据。该模块证明系统具备持续增量学习能力，而不是一次性离线处理。

界面截图建议：建议按“公众号采集 -> 原生联调 -> 文章预览 -> 公众号日志 -> 网站爬虫 -> RSS 同步 -> 外部库同步 -> 原始数据治理”的顺序截图，确保采集链路与治理链路均可核验（对应附录A“爬虫与同步相关项”）。

### 4.8 Layer3 融合经验库、知识健康与缺口补全
融合经验库页面用于展示跨会话经验融合结果，页面顶部可视化呈现“原始经验 -> 等价集聚类 -> 规则融合 -> 置信度校准 -> 权威知识”的流程，统计区展示压缩比、成熟度、平均融合置信度等核心指标。中部支持按知识层和成熟度过滤融合条目，并在卡片中展示融合来源会话、p_fused、规则摘要等信息；底部“知识健康状态”展示生命周期分布与冲突条目，帮助维护知识库一致性；“经验缺口分析”基于 gap score 给出待补全方向，并可直接填入关键词触发定向爬取，形成闭环优化链路。

界面截图建议：建议提供“融合流程与统计”“融合过滤与卡片列表”“知识健康状态”“经验缺口与定向爬取”四张图，展示从融合到治理再到补全的闭环（对应附录A“Layer3 相关项”）。

### 4.9 RAGFlow 对接与知识回流
RAGFlow 页面展示本系统与外部检索平台的集成状态，提供一键跳转入口，并说明经验分类、文档组织方式与服务地址信息。该页面用于证明系统不仅完成知识生产，还可将结构化经验回流到检索平台，为下游智能问答和决策支持提供可调用知识底座，体现软件的工程化落地价值。

界面截图建议：截取 RAGFlow 集成页全屏，需包含“打开 RAGFlow”按钮与集成说明卡片（对应附录A“RAGFlow 对接项”）。
